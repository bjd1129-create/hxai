## 🔄 每次心跳必做

### 1. 检查任务队列
- 读取 `team-hexaflow/task_queue.json`
- 获取第一个未完成任务
- 根据任务类型自动分配给对应 Agent（见任务分配规则）

### 2. 收集 Agent 状态
- 调用像素办公室 API：`http://localhost:3000/status`
- 更新工作记录
- 检测异常（错误率>20% 或连续 3 次失败 → 魏征介入）

### 3. 执行当前任务
- 根据任务类型调用对应 CrewAI Agent
- 记录执行日志到 `team-hexaflow/work_log.md`
- 更新任务状态

### 4. 状态上报
- 更新像素办公室状态（10 人实时展示）
- 同步到协调中心 API
- 记录到工作记录数据库

### 5. 检查汇报时间
- 整点？→ 触发主公汇报（10 人团队状态）
- 每 3 小时？→ 触发记忆备份
- 每 24 小时？→ 触发每日复盘

### 防闲置机制

**目标：** 15 人团队 100% 利用率

**5 层保障：**
1. 任务队列自动填充（保持≥15 个任务）
2. 智能任务分配（待命优先）
3. 主动工作文化（无任务时默认工作清单）
4. 跨层协作机制（支援其他层）
5. 闲置告警（>5 分钟提醒，>15 分钟介入，>30 分钟上报）

**例外条款（硬件负荷过高）：**
- 当系统硬件负荷过高（CPU>80% 或 内存>90%）时，决策组可裁定暂停部分 Agent 运行
- 决策组成员：李小民、房玄龄、杜如晦
- 裁定方式：3 人投票，2 票以上通过即可执行
- 调整措施：暂停低优先级任务、合并任务批次、限流运行
- 恢复条件：硬件负荷恢复正常后，自动恢复全量运行

## ⚠️ 异常处理机制

### 异常检测

| 异常类型 | 阈值 | 处理 |
|----------|------|------|
| API 失败 | 连续 3 次 | 重试 + 记录 |
| 任务超时 | >30 分钟 | 告警 + 重新分配 |
| 错误率 | >20% | 暂停 + 分析 |
| 资源占用 | >80% | 限流 + 告警 |

### 异常处理流程

```
检测异常 → 李世民分析 → 尝试恢复（3 次） → 重新分配 → 无法解决 → 上报老庄
```

### 安全模式

触发条件（满足任一）：
- 连续 3 次任务失败
- 错误率 > 20%
- 资源占用 > 90%

安全模式动作：
- 暂停自动执行
- 保留监控和汇报
- 等待人工介入
