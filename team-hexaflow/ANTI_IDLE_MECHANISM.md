### 闲置根本原因
1. **任务队列不足** - P0/P1 任务太少
2. **任务分配不均** - 某些职能任务过多，某些过少

4. **技能未充分利用** - 成员多技能，但只使用单一技能

### **第 2 层：智能任务分配**

**机制：** 根据成员状态自动分配，优先级：**待命 > 空闲 > 低负载**

```python
# 任务分配优先级
def assign_task(task):
    # 1. 优先分配给待命成员
    idle_members = [m for m in team if m.status == 'idle']
    if idle_members:
        return best_match(idle_members, task)
    
    # 2. 其次分配给低负载成员（进度<50%）
    low_load = [m for m in team if m.progress < 50]
    if low_load:
        return best_match(low_load, task)
    
    # 3. 最后加入队列
    task_queue.append(task)
```

**分配规则：**
| 成员状态 | 定义 | 分配优先级 |
|----------|------|------------|
| 🟢 工作中 | progress > 50% | 低（除非紧急） |
| 🟡 思考中 | 10% < progress < 50% | 中 |
| ⚪ 待命中 | progress = 0% 或无任务 | **最高** |

### **例外条款：硬件负荷过高**

**触发条件：** CPU>80% 或 内存>90%

**决策组：** 李小民、房玄龄、杜如晦（3 人）

**裁定方式：** 3 人投票，2 票以上通过即可执行

**调整措施：**
- 暂停低优先级任务（P2/P3）
- 合并任务批次
- 限流运行（50% 负载）

**恢复条件：** 硬件负荷恢复正常后，自动恢复全量运行

## 🚀 **执行流程**

### 每次心跳检查（5 分钟）

```python
def heartbeat_check():
    # 1. 检查任务队列
    if len(task_queue) < 15:
        auto_fill_tasks()
    
    # 2. 检查成员状态
    for member in team:
        if member.status == 'idle' and member.idle_time > 10:
            # 3. 分配任务
            if not assign_task(member):
                # 4. 建议主动工作
                suggest_auto_task(member)
    
    # 5. 闲置告警

        send_alert("闲置人员过多，请李小民介入")
```
